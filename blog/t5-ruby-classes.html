Why and How To Use the Ruby Class Structure

3/28/15

Like I’ve said before, as programmers we should always be looking for ways to decrease the amount of work we have to do when writing code, especially when it comes to manually typing repetitive code over and over. Luckily, Ruby has a very helpful way to organize and reduce code. The class structure is a great tool for creating and defining methods and attributes for multiple objects that share the same properties.

Let’s say we want to create a type of object in Ruby called “Dessert_option.” For each Dessert_option you come up with, rather than repeatedly identify the properties of each one, you can define a Dessert_option class that will automatically populate each of the Dessert_option objects with the specified methods. For example, you probably know that each Dessert_option has a “name.” Maybe you’d also like to specify that each has a “location”, a “specialty”, a “quality”, or anything else you choose! You can also choose whether you’d like each of these properties to be readable (where you can output the value that is already determined), writable (where you can input the value on the spot), or both! Our class definition might look something like this:

class Dessert_option

attr_reader :quality

attr_accessor :location, :specialty, :price_range

def initialize

@quality = “Excellent”

@location

@specialty

@price_range

end

def specialty(dessert)

@specialty = dessert

end

end

Properties like location and specialty can be input with the creation of each new Dessert_option. Notice that in this case the quality is predetermined to be “Excellent” regardless of which option you’re talking about, by nature of these all being desserts…
